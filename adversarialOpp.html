Adversarial Techniques for Cyber Infiltration

Introduction

This page discusses how cyber adversaries can leverage distractions and manipulate circumstances to infiltrate target networks. Here's a breakdown of the key points:

Adversarial Techniques

  • DDoS Attacks as Cover: Attackers can use DDoS attacks to overwhelm security systems, hindering their ability to detect other malicious activity on the network.
  • Exploiting Service Disruptions: Adversaries can target service providers or infrastructure to create disruptions that distract security teams and potentially open windows for infiltration.
  • Patch and Update Vulnerabilities: Attackers might wait for the release of faulty patches or updates to exploit vulnerabilities created during troubleshooting processes.
  • Infiltrating During Acquisitions: Mergers and acquisitions can create confusion and introduce new systems that are not yet fully secured, making them vulnerable targets.
  • Physical Access at Events: Attackers can blend into crowds at conferences or vendor events to gain physical access to a target network.

Defensive Strategies

  • Layered Security: Implementing multiple layers of security controls can make it more difficult for attackers to exploit any single weakness.
  • Security Awareness Training: Train security personnel to be aware of these adversarial tactics and to remain vigilant during disruptive events.
  • Threat Modeling and Risk Management: Proactively identify potential attack scenarios and develop mitigation strategies.
  • Change Management Protocols: Establish clear procedures for handling system changes and updates to minimize security risks during these processes.
  • Penetration Testing: Conduct regular penetration testing exercises to identify vulnerabilities in security controls and incident response capabilities.
  • Contingency Planning: Develop plans for responding to security incidents that occur during disruptive events.

Security Controls and Mitigations (NIST 800-53)

  • AC-14: Permitted Actions Without Identification or Authentication: Define acceptable security measures for emergencies when standard authentication protocols may be bypassed.
  • AC-20: Use of External Information Systems: Establish policies for using external systems, especially during critical situations, to prevent unauthorized access.
  • CA-8: Penetration Testing: Conduct penetration testing exercises that simulate real-world attack scenarios, including those that leverage distractions.
  • CP-2: Contingency Plan: Integrate adversary awareness and threat hunting practices into contingency plans for various disruptive events.
  • CP-3: Contingency Training: Include penetration testing during contingency training to assess security measures during emergencies.
  • CP-7: Alternate Processing Site: Evaluate the security risks associated with resuming operations at backup sites and implement safeguards.
  • IR-2: Incident Response Training: Train security personnel to identify and respond to security incidents, including those that occur during disruptions.
  • PE-3: Physical Access Control: Enforce stricter physical access controls during situations with a high volume of visitors or limited oversight.
  • PE-10: Emergency Shutoff: Develop a capability to shut down information systems if a security compromise is suspected during an emergency.

Conclusion

By understanding these adversarial tactics and implementing appropriate security measures, organizations can significantly improve their ability to defend against cyberattacks, even during disruptive events.
